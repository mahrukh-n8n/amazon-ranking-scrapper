# Requirements: Amazon Geo-Rank Scraper

**Defined:** 2026-01-30
**Core Value:** Accurate Geo-Rank extraction validation (Zip Code persistence).

## v1 Requirements

Requirements for initial release.

### Core Scraper
- [x] **SCRAPE-01**: System can successfully set Amazon Delivery Zip Code via UI interaction.
- [x] **SCRAPE-02**: System verifies "Deliver to" location matches target Zip Code before scraping.
- [x] **SCRAPE-03**: System pauses and alerts user (sound/focus) when Captcha is detected.
- [x] **SCRAPE-04**: System persists session cookies to reuse location settings across runs.
- [x] **SCRAPE-05**: System runs in "Headed" mode (visible browser) by default.

### Data Management
- [x] **DATA-01**: User can load job files (Excel/CSV) with columns: Marketplace, Zip Code, ASINs, Keywords.
- [x] **DATA-02**: System exports results (Rank, Page, ASIN, Keyword, Date) to CSV/Excel.
- [x] **DATA-03**: (Optional) System can push results via HTTP POST to a webhook URL (e.g., n8n).

### User Interface
- [x] **UI-01**: User can Start, Pause, and Stop the scraping job.
- [x] **UI-02**: User sees real-time activity log (e.g., "Setting Zip to 10001...", "Searching 'shoes'...").
- [x] **UI-03**: System displays desktop notification when job is complete.

## v2 Requirements

Deferred to future release.

- **DATA-04**: Screenshot Evidence (Capture image of rank position).
- **UI-04**: Visual Dashboard/Charts of rank history.
- **CORE-06**: Proxy rotation integration (API-based).

## Out of Scope

| Feature | Reason |
|---------|--------|
| Headless Execution | Browser stealth requires visible rendering; user wants verification. |
| Automated Captcha Solving | High cost/complexity; manual intervention preferred. |
| Server Deployment | Desktop-first design; "Deliver to" logic fragile on server IPs. |
